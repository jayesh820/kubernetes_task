Case Studies

Booz Allen Hamilton — Federal Modernization (recreation.gov)

Rebuilt a legacy federal booking platform into microservices on Kubernetes, enabling faster changes per agency and improved reliability.

Outcome: Change implementation dropped from days to minutes; centralized ingress and security simplified operations at scale.

Booking.com — Native Kubernetes Platform

Shifted from a heavy distro to vanilla Kubernetes, training engineers on native APIs and patterns.

Outcome: New service setup in minutes; hundreds of services, ML inference workloads, and frequent safe releases.

Decision Guide

  • Choose K8s when you need scale, automation, portability, or internal platform consistency.
  • For simple, stable apps, consider PaaS/VM first; adopt K8s as complexity grows.
  • Prereqs: CI/CD, observability, container maturity, SRE on-call readiness.

Operational Snippets

Deployment with Probes + Resources 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web
spec:
  replicas: 3
  selector: { matchLabels: { app: web } }
  template:
    metadata: { labels: { app: web } }
    spec:
      containers:
      - name: app
        image: ghcr.io/acme/web:1.2.3
        ports: [{ containerPort: 8080 }]
        resources:
          requests: { cpu: "250m", memory: "256Mi" }
          limits:   { cpu: "750m", memory: "512Mi" }
        readinessProbe:
          httpGet: { path: /healthz, port: 8080 }
          initialDelaySeconds: 5
          periodSeconds: 5
        livenessProbe:
          httpGet: { path: /livez, port: 8080 }
          initialDelaySeconds: 10
          periodSeconds: 10
Horizontal Pod Autoscaler 
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: web-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: web
  minReplicas: 3
  maxReplicas: 20
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
Service + Ingress (TLS) 
apiVersion: v1
kind: Service
metadata:
  name: web
spec:
  selector: { app: web }
  ports: [{ name: http, port: 80, targetPort: 8080 }]
---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: web
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt
spec:
  tls:
  - hosts: [ "app.example.com" ]
    secretName: web-tls
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: web
            port: { number: 80 }

FAQs

Is K8s overkill for small apps? Sometimes—managed PaaS can be faster until complexity demands K8s.

Does K8s help ML? Yes—predictable environments and autoscaling for inference/batch.

Is multi-cloud realistic? With discipline (GitOps, portable services) it’s very achievable.
